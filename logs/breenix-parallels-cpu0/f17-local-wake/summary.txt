## Determination

H1 is falsified: IRQ return calls the resched path. Both diagnostic-run2 and final run3 show `IRQ_TAIL_CHECK_RESCHED` followed by `RESCHED_CHECK_ENTRY`.

H2 is falsified: the resched path drains the wake buffer before the scheduling decision. Both diagnostic-run2 and final run3 show `RESCHED_CHECK_DRAINED_WAKE` with `wake_success=1` before `RESCHED_CHECK_SWITCHED`.

Confirmed bucket: Other. The local wake path completes and switches to the woken task. The later failure is an AHCI/SPI34 condition: the timeout dump reports `GICD_ISPENDR[1]` pending for SPI34, `GICD_ISACTIVER[1]` active for SPI34, `AHCI_PORT1_IS=0x1`, `CI=0x0`, and a later command number (`cmd#=1374`) than the completed local wake (`last_port1_cmd_num=1372`).

## Verdict

VERDICT: FAIL. Four of five final runs met the pass criteria. Run3 reached `[init] bsshd started (PID 2)` but later had two AHCI port timeouts and soft lockup dumps. No corruption markers were observed.

Recommended F18 direction: investigate why SPI34 remains active/pending with AHCI Port1 `IS=0x1` and no completion for a later command after the local wake path has already switched successfully. This likely belongs in the AHCI interrupt EOI/deactivation or command publication/completion corridor, not the local IRQ-return scheduler drain path.
